ABOUT ME
전하영
교사에서 AI까지, 문제를 정의하고 시스템으로 풀어내는 엔지니어

4년간의 교직 생활은행정 현장의 비효율을 정의하고 현장에서 직접 적용할 시스템을 구축하는 실험의 장이었습니다. 출결 자동화 웹앱을 직접 만들어 배포하고, 학교 홈페이지의 인프라와 보안을 관리하였으며, 학생 상담을 위한 데이터를 분석하며 데이터의 관계성을 파악하고 시스템을 구축해왔습니다.

이 과정에서 ChatGPT, Claude, Gemini 등 생성형 AI 도구를 업무에 직접 적용하며 데이터와 AI를 본격적으로 학습할 계기가 되었습니다. SK네트웍스 AI 부트캠프에서는 Python, RAG, Agent까지 End-to-End로 학습하고 팀 프로젝트를 수행하며, 데이터 수집에서 평가까지 전 과정을 직접 수행한 파이프라인 역량을 체득했습니다.

데이터 이면의 인과관계를 끝까지 추적하는 분석력과 교사 경력에서 단련된 사람 관찰력, 갈등 조정 경험을 바탕으로 팀 내에서 문제의 본질을 먼저 정의하고각방의 의견을 경청하여 합의점을 이끄는 역할을 합니다.

01
옆집 손주
GraphRAG · LangGraph · Memgraph · RAGAS 평가
RAGAS 평가 결과 Faithfulness 0.89, Answer Relevancy 0.91
실제 독거노인 10명과 시범 적용하여 법률 용어 이해도 40% 향상
GraphRAG 기반 노인 법률 상담 AI 에이전트입니다. Memgraph 기반 그래프 DB로 법률 조문 간 관계를 모델링하고, LangGraph로 다단계 상담 플로우를 제어합니다. 독거노인의 법률 소외 문제를 해결하기 위해 시작했습니다.
주요 구현
  • Memgraph GraphRAG로 법률 조문 관계 그래프 구축
  • LangGraph 기반 다단계 상담 에이전트 플로우 설계
  • RAGAS로 검색 응답 품질 정량 평가
  • 대화 히스토리 기반 상황 인식 응답
WHY
독거노인의 법률 소외 문제 해결, 복잡한 법률 문서를 일반인도 이해할 수 있게 만들기
HOW
GraphRAG로 법령 간 관계 그래프 모델링, LangGraph로 상담 에이전트 플로우 구현, RAGAS 지표로 품질 평가
WHAT
LangGraph 상담 플로우 설계 및 구현, Memgraph 그래프 DB 구축, RAG 파이프라인 + RAGAS 평가 지표 설계
트러블 슈팅
그래프 DB의 관계성 모델링부터 RAGAS 평가까지 End-to-End로 경험하며 RAG 서비스 구축의 전 과정을 체득.
성장한 점
RAG 구축-운영, Vector/Graph DB 설계, LLM 에이전트 설계, RAGAS 평가 역량.

CEO 경영 시뮬레이터
XGBoost · React 19 · 의사결정 예측 시뮬레이션
프로젝트 전시회에서 200명 이상이 체험, 경영 기초 이해도 평균 30% 향상
사용자가 경영 의사결정을 내리고 그에 따른 결과를 예측 시뮬레이션할 수 있는 인터랙티브 게임입니다. XGBoost로 경영 성과 예측 모델을 학습시키고, 의사결정 트리와 연결하여 시뮬레이션 결과를 제공합니다.
주요 구현
  • XGBoost 기반 경영 성과 예측 모델
  • 시나리오별 의사결정 분기 트리
  • 실시간 재무제표 시각화
  • React 19 기반 인터랙티브 UI
WHY
경영학 이론을 실전에서 체험하고 싶었음, 데이터 기반으로 의사결정이 이루어질 때 결과가 어떻게 달라지는지 궁금
HOW
과거 기업 경영 데이터로 XGBoost 모델 학습, 의사결정 트리와 연결하여 시뮬레이션 결과 예측
WHAT
예측 모델 설계 및 학습, 의사결정 트리 로직 구축, React 19 기반 프론트엔드 UI 개발
트러블 슈팅
ML 모델을 웹 서비스에 통합하는 과정에서 백엔드-프론트엔드-데이터 사이언스의 경계를 넘나드는 경험을 했다.
성장한 점
XGBoost 모델링, 의사결정 시스템 설계, React 19 프론트엔드 개발.

03
고객 이탈 예측
앱 리뷰 크롤링 38.9만건 · ML 모델링 · AUC-ROC 평가
팀 내 유일하게 인과관계를 이끌어내며 차별화된 인사이트 제공
모델 친화적 피처 엔지니어링 능력 돋보임 - 멘토 -
국내 이커머스 고객 이탈 예측 프로젝트입니다. 공식 데이터 부재로 앱스토어/플레이스토어 리뷰 38.9만건을 크롤링하여 유의미한 데이터셋을 구축하고, ML 모델링부터 평가까지 전 과정을 수행했습니다.
주요 구현
  • 앱스토어 + 플레이스토어 리뷰 크롤링 (38.9만건)
  • 텍스트 데이터 감성 분석 및 불만 요인 정량화
  • 로그 변환, 인코딩 전략, 샘플링 기법 적용
  • AUC-ROC, PR 곡선, 혼동행렬로 다각도 평가
WHY
국내 이커머스 공식 데이터 부재, 앱 리뷰가 고객 이탈 징후와 페인 포인트를 가장 직접적으로 보여주는 원천임을 발견
HOW
크롤링 → EDA → 전처리 → 모델링 → 평가, 모든 선택에 기술적 근거 수립
WHAT
데이터 수집 파이프라인 설계, EDA 및 전처리, 모델 선정, 하이퍼파라미터 튜닝, 정량적 평가 레포트 작성
트러블 슈팅
국내 데이터 부재 → 앱 리뷰 크롤링으로 대안 데이터 확보. 카테고리형 데이터 한계 → 변수 간 상관관계 분석으로 인코딩 전략 최적화.
성장한 점
모든 선택에 기술적 근거를 수립하는 것의 중요성. 데이터 이면의 맥락을 읽어내는 분석력.

주차장 데이터 정제 파이프라인
Selenium + Pandas · 주소 정규화 · 데이터 무결성 검증
수집 단계의 노이즈가 최종 분석 결과에 직접적 영향을 미친다는 것을 체감
정제 로직의 완성도가 서비스 전체 신뢰도를 결정함을 확인
네이버 지도 기반 주차장 데이터를 수집 정제하여 분석용 DB를 구축하는 프로젝트입니다. 도로명 주소와 지번 주소가 혼재된 데이터를 정규화하고, 텍스트 노이즈를 제거하는 예외 처리 파이프라인을 설계했습니다.
주요 구현
  • Selenium으로 네이버 지도 데이터 크롤링
  • Pandas로 도로명/지번 주소 체계 표준화
  • 예외 처리 파이프라인으로 노이즈 제거
  • 반복적 대조 코드로 데이터 무결성 검증
WHY
도로명 주소와 지번 주소가 무분별하게 혼재, 분석을 위해서는 통일된 주소 체계 필요
HOW
Selenium 자동화로 데이터 수집, Pandas로 주소 체계 표준화 정제 로직 설계, 대조 코드 반복 실행하며 무결성 검증
WHAT
정제 파이프라인 설계 및 구현, 예외 처리 로직 구축, 무결성 검증 자동화
트러블 슈팅
전처리 이후에도 데이터 유실 중복 발생 → 반복적 대조 코드 실행으로 무결성 검증. 수집 단계의 품질이 전체 품질을 결정.
성장한 점
데이터 정제의 완성도가 분석 결과의 신뢰도를 결정한다는 것을 실전에서 체득.
